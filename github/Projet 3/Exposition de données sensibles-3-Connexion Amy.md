# Exposition de données sensibles-3-Connexion Amy

## 1. Méthodologie

   1. Consulter le site et fouiller les pages pour trouver des indices sur "Amy".  
   2. Découvrir que "Amy" est une réference et que son mari est "kif".
   3. taper sur google la citation 93.83 billion trillion trillion centuries.
   3. allez a cette adresse  https://www.grc.com/haystack.htm. 
   4. deduire le mot de passe de amy qui est pas trés fute.
   4. Entrer l’adresse mail dans la page de connexion.  
   5. Exploit réussi : accès au compte Amy grâce aux informations découvertes.  

**techniques utilisées :**

   - Exploitation d’informations publiques / manque de protection des données

## 2. Vulnérabilités

- **type :** Mauvaise gestion des informations sensibles  
- **composant affecté :** Système d’authentification  
- **Séverité estimée :** Élevée  

## 3. Risques

- Connexion à des comptes tiers  
- Accès aux informations personnelles des utilisateurs  
- Fuite de données  
- Mauvaise réputation  
- Perte de confiance  

## 4. Actions

- **Mitigation :**  
   - Éviter de divulguer des informations sensibles dans le contenu public.  
   - Masquer les adresses mails des utilisateurs.  

- **Correctifs :**  
   - Restreindre l’accès aux données personnelles.  
   - Ajouter une authentification multi-facteurs.  

- **Bonnes pratiques :**  
   - Effectuer des tests réguliers de sécurité.  
   - Suivre les recommandations OWASP pour l’authentification.
